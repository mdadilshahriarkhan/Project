Common Git Commands

What is Git:
Git is a Open-souece and free "Version Control System" too that helps to track changes in code.

What is GitHub:
GitHub is a Website (Cloud-based platfrom) that allows developers to store and manage their code using Git

Configuring user information, initializing and cloning repositories
# git init

"Status" Check
# git status

"add" command (Adding files to staging area)
# git add [filename]
# git add *
# git add . 			

"commit" (It is used to save the changes from the staging area to Git Repository)
Each commit has a Unique Identifier (SHA-1 hash).
# git commit -m "[message]"	

"log" command (Viewing commit History)
# git log
# git log --oneline 

"diff" differences between the last 2 commits in Git. Its usefull for Code reviews and Debugging.
It show the differences between the Working directory and the Staging Area.
# git diff
# git diff <commit1> <commit2> <commit3>

"clone" (Create a copy of an existing Git Repository from remote server like: Github, GitLab to our local machine)
# git clone [Repository-url]

"fetch" (Retrieve the latest updates from a remote repository without Merging them into our local branch)
# git fetch <remote-repositoriy-name/url>

"Merge" (Merge the changes between different local repositories or branches)
# git merge FETCH_HEAD

"pull" (git pull = git fetch + git merge)
# git pull <repositories-url> <branch>
# git pull <remote> <branch>
# git pull 

"push" (updates local commits to a remote repository)
# git push <repositories-url> <branch>

WORKFLOW: 
GitHub repo > Clone > changes > Add > commit > push 

Go to your project folder
# cd your-project-folder
Step-1: git init
Step-2: git add . / git add [filename]
step-3: git commit -m "first commit"
step-4: open GitHub panel and create a Repository (Repository name : Database)
step-5: git remote add origin https://github.com/your-username/your-repo.git
		git remote add origin https://github.com/mdadihahriarkhan/Database.git
Step-6: git branch -M main 			// Set branch name
Step-7: git push -u origin main		// Push to GitHub


A typical Git workflow involves several core commands: 
# git init								: Initializes a new Git repository in a local directory.
## git clone [Repository-url]			: Creates a local copy of a remote repository
# git add [file]						: Stages changes for the next commit.
# git commit -m "message"				: Records the staged changes as a snapshot in the project history with a descriptive message
# git branch -M main					: change branch ("main" is Github default branch)
# git remote -v							: Check remote connection

# Open "GitHub" > create a repository name: localproject > and connect with remote using below link 
# git remote add origin https://github.com/mdadihahriarkhan/localproject.git 

# git push -u origin main				: Uploads local commits to a remote repository (e.g., on GitHub).
# git pull								: Updates the local repository with the latest changes from the remote counterpart.
